# Pacific Tourist Arrivals Visualization

## Audience

TBD. Assume a general workshop or public-interest audience until the user chooses a sharper audience.

## Purpose

Create an HTML visualization about tourist arrivals across Pacific Island countries and territories using the supplied SPC climate-change indicators CSV.

With the user-supplied precipitation anomalies dataset, the project may broaden into a tourism-and-climate-context artifact. The precipitation layer should be used as context unless the analysis supports a stronger claim.

Selected direction: Climate Context Dashboard with a comparative narrative frame.

## Personality

Clear, evidence-led, and approachable. The artifact should make gaps and uncertainty visible rather than hiding them.

## Anti-References

- Avoid implying that every country has the same data coverage.
- Avoid ranking destinations without explaining that raw arrivals are not normalized by population, land area, tourism capacity, or flight access.
- Avoid treating missing years as zero arrivals.
- Avoid implying that annual precipitation anomalies caused tourist-arrival changes.

## Design Principles

- Prioritize time coverage and country comparability.
- Keep raw-count claims separate from normalized or contextual claims.
- Make pandemic-era disruption visible if the data supports it.
- Use climate anomaly data as a visible companion layer when it clarifies context, and label it carefully as context rather than causation.
- Make country-to-country comparison easy without hiding missing tourist-arrivals years.

## Accessibility Goals

- Use readable labels, high-contrast color, and non-color encodings for important categories.
- Include caveats in visible text, not only hover states.
